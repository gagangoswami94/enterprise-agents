# Examples

Multi-agent workflow examples showing how to coordinate specialists from Enterprise Agents on real tasks.

## What These Show

Single agents are useful. Teams of agents are transformative. These examples demonstrate how to chain specialists — passing outputs as inputs, running work in parallel, using quality gates — to accomplish complex objectives faster than any single generalist could.

## Patterns Used Across All Examples

**Sequential handoff** — Agent A's output becomes Agent B's input. Always paste the full output, not a summary.

**Parallel tracks** — Independent workstreams run simultaneously (e.g., copy and design, backend and frontend) and merge at a defined point.

**Quality gates** — Reality Checker or Compliance Auditor reviews work before the next phase begins. A NO-GO sends you back to the responsible specialist, not to start over.

**Context passing** — Agents don't share memory by default. Pass prior outputs explicitly in each new activation prompt.
